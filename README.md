# SMS_SPAM_DETECTION

Project Overview

This project implements an end-to-end machine learning pipeline to classify SMS messages as Spam or Ham using ensemble learning techniques. The work is aligned with the syllabus topics Combining Classifiers, Boosting, Decision Stumps, and K-Fold Cross Validation.

Objective

To build, evaluate, and compare:

Individual base classifiers

Combined classifiers (Voting and Stacking)

A boosting model using AdaBoost with decision stumps

and to identify the most effective model for SMS spam classification.

Dataset

Source: UCI Machine Learning Repository

Dataset: SMS Spam Collection

File Used: SMSSpamCollection

Format: Tab-separated text file

Columns

label – spam or ham

message – SMS text

Methodology
1. Data Preprocessing

Loaded the dataset using Pandas

Removed missing values

Encoded class labels (ham = 0, spam = 1)

Converted text messages into numerical features using TF-IDF Vectorization

2. Base Learners

The following individual classifiers were implemented:

Multinomial Naive Bayes

Logistic Regression

Linear Support Vector Machine (SVM)

3. Ensemble Techniques

Hard Voting Classifier

Soft Voting Classifier

Stacking Classifier

Base learners: Naive Bayes, Logistic Regression

Meta-learner: Logistic Regression

4. Boosting

AdaBoost Classifier

Weak learner: Decision Tree with max_depth = 1 (decision stump)

5. Model Evaluation

Stratified K-Fold Cross Validation (k = 5)

Evaluation metrics:

Precision

Recall

F1-score

ROC-AUC (computed only for probabilistic models)

Note: ROC-AUC is not applicable to Linear SVM and Hard Voting classifiers as they do not provide probability estimates.

Output Files
1. ensemble_comparison.csv

This file contains the model-wise comparison of all classifiers using mean performance metrics across 5 folds.

Columns:

Model

Precision_mean

Recall_mean

F1_mean

ROC_AUC_mean

2. final_model_predictions.csv

This file contains predictions generated by the final selected model (Soft Voting Classifier).

Columns:

MessageId

Actual

Predicted

Probability

Final Recommendation

Based on Stratified 5-Fold Cross Validation, the Soft Voting Classifier achieved the best balance of precision, recall, F1-score, and ROC-AUC. Soft voting effectively combines probabilistic outputs from multiple base learners, resulting in stable and robust performance. Therefore, Soft Voting is recommended as the most effective combining strategy for SMS spam classification.
